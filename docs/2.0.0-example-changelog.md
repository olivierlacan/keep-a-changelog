# Keep a Changelog 2.0: worked example

A complete, self-consistent example changelog that applies the 2.0 guidance in
practice. The project (`acme/widget`, a small HTTP client library) is fictional
but coherent: every entry hangs together so the lifecycle reads realistically.

House constraints are honored: no em-dashes or en-dashes, only the six types,
every version carries a reference-style compare link, and entries are written for
the people who use the software rather than as reworded commit messages.

## The example

```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/2.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).
Versioning covers the public library API (the exported functions and their options).

## [Unreleased]

### Added

- `Client.stream()` for reading large responses without holding them in memory.

## [2.0.0] - 2026-05-20

This release stabilizes the request API and removes the legacy callback interface deprecated in 1.4.0. Most projects only need to rename one option to upgrade; one larger change has a migration guide.

### Added

- Automatic retries for failed idempotent requests, with exponential backoff. Configure it with the new `retry` option.
- `Client.withTimeout(ms)` to set a per-request timeout.

### Changed

- **Breaking:** `request()` now returns a `Response` object instead of the raw body. Read the body with `response.body()`. See the [2.0 migration guide](https://github.com/acme/widget/blob/main/docs/migrate-2.0.md) for the full upgrade.
- **Breaking:** renamed the `headers` option to `defaultHeaders` to make its scope clear. Rename the option in your client setup.
- Connection pooling is on by default. Requests to the same host now reuse connections, so a busy client opens fewer sockets.

### Deprecated

- `Client.setProxy()` is deprecated in favor of the `proxy` option passed to the constructor. It will be removed in 3.0.0.

### Removed

- **Breaking:** removed the callback-style `request(url, callback)` form deprecated in 1.4.0. Pass a URL and await the returned promise instead.

### Fixed

- Requests no longer hang when the server closes a connection mid-response. The client now surfaces a `ConnectionClosed` error.

### Security

- CVE-2026-31840: redirects to a different host no longer forward the `Authorization` header, preventing credential leaks to untrusted destinations.

## [1.4.0] - 2026-02-11

### Added

- Promise-based `request()` that returns the response body. This is the supported form going forward.

### Deprecated

- The callback-style `request(url, callback)` form is deprecated in favor of the promise-based form. It will be removed in 2.0.0.

### Fixed

- Query strings with repeated keys are now encoded correctly instead of keeping only the last value.

## [1.3.1] - 2025-11-30 [YANKED]

This release was yanked: the gzip fix below corrupted responses larger than 64 KB. Use 1.3.2 instead.

### Fixed

- Gzip-encoded responses are now decompressed instead of returned as raw bytes.

## [1.3.0] - 2025-11-18

### Added

- Support for `gzip` and `deflate` response compression.

### Changed

- The default request timeout is now 30 seconds instead of unlimited, so a stalled server no longer blocks a request forever.

[Unreleased]: https://github.com/acme/widget/compare/v2.0.0...HEAD
[2.0.0]: https://github.com/acme/widget/compare/v1.4.0...v2.0.0
[1.4.0]: https://github.com/acme/widget/compare/v1.3.1...v1.4.0
[1.3.1]: https://github.com/acme/widget/compare/v1.3.0...v1.3.1
[1.3.0]: https://github.com/acme/widget/releases/tag/v1.3.0
```

## What this demonstrates

- **`## [Unreleased]` at the top:** kept short with one accrued entry, showing the
  convention without padding. Its link compares the latest tag to `HEAD`.
- **Dated heading + optional summary:** `## [2.0.0] - 2026-05-20` uses
  `YYYY-MM-DD`, and the one-sentence summary earns its place because a major
  release with breaking changes is worth introducing. Smaller releases (1.4.0,
  1.3.0) skip the summary, showing the judgment call.
- **All six types, curated and user-facing:** every entry is written for someone
  using the library, not as a reworded commit. The preamble states what the
  versioning scheme covers (the public library API), as the Breaking-changes
  section asks.
- **Fixed vs Changed:** the 2.0.0 *Fixed* entry (requests no longer hang) was a
  genuine bug. The 2.0.0 *Changed* entry (pooling on by default) and the 1.3.0
  *Changed* entry (timeout now 30s) were intentional behavior that now works
  differently. Same contrast in 1.4.0/1.3.1: query-string encoding and gzip were
  bugs, so they are *Fixed*.
- **Security leads with the CVE:** `- CVE-2026-31840: ...` so tools and readers
  can match the advisory.
- **`**Breaking:**` kept inside its type, with inline note and a link-out:** the
  renamed-option break stays in *Changed* with an inline upgrade note; the larger
  `request()` return-type change is also in *Changed* but links out to a migration
  guide because the steps are substantial; the removed callback form carries
  `**Breaking:**` inside *Removed*. No separate "breaking changes" section.
- **Deprecate-then-remove discipline, visible across versions:** the callback form
  is marked *Deprecated* in 1.4.0 naming its removal version (2.0.0), then appears
  under *Removed* in 2.0.0. A second live deprecation (`Client.setProxy()`) names
  3.0.0, showing the pattern forward in time.
- **Reference-style links for every version:** each points at a `compare` URL
  except the oldest, which points at its tag (nothing earlier to compare against),
  keeping links out of the headings.
- **Lifecycle across versions, including a yank:** a minor (1.4.0), a patch
  (1.3.1, marked `[YANKED]` with a plain-language reason and a pointer to the
  fixed release), and an earlier minor (1.3.0) make the compare-link chain
  realistic.
