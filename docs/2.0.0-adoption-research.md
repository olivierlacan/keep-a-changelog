# Keep a Changelog — worldwide reach & adoption (research)

Background research gathered for the 2.0.0 References section: hard, citable,
auditable numbers on how widely Keep a Changelog is used.

**Observed: 2026-06-05.** Primary sources preferred (GitHub REST API, npm/PyPI/
crates.io download APIs, the project repo). Each figure is labelled exact /
estimate / lower-bound with a source and date. Nothing here is fabricated; gaps
are stated as gaps.

> **Staleness warning.** Every GitHub and download figure grows or shifts over
> time. If any of these go onto the site, cite them with the observation date and
> re-pull right before publishing. The **28 languages** figure (from `config.rb`)
> is the only one safe to state flatly.

semantic-release, changesets, conventional-changelog, release-please, git-cliff
and towncrier are changelog-*generation* tools — adoption signals for the category
KAC defined, not direct KAC parsers. The two **direct KAC parsers** are
`keepachangelog` (PyPI) and `keep-a-changelog` (npm). GitLab's own changelog
tooling uses git-trailer generation and does **not** reference KAC — do not claim
GitLab endorsement.

### 4a. Top-popularity sweep (header verified live, 2026-06-08)

Re-ran the check against the **83 most-starred genuine software projects** on
GitHub (star range ~37K–245K; curricula, awesome-lists, books, and public-apis
excluded). The result confirms the finding above rather than overturning it:
**only 3** of the 83 carry an explicit KAC reference. The rest either keep a
non-KAC changelog (19) or publish history through GitHub Releases / docs (61).
The takeaway stands: the headline is the 20,640-file aggregate, not a marquee
roster. Still, these three are worth recording because they are verified and one
(open-webui) is a very high-star project.

| Project | Stars | KAC version | CHANGELOG (header verified) |
|---|---|---|---|
| open-webui/open-webui | 140,703 | 1.1.0 | `raw.githubusercontent.com/open-webui/open-webui/main/CHANGELOG.md` |
| Kong/insomnia | 38,470 | 1.0.0 | `raw.githubusercontent.com/Kong/insomnia/master/CHANGELOG.md` |
| ajeetdsouza/zoxide | 37,299 | 1.0.0 | `raw.githubusercontent.com/ajeetdsouza/zoxide/main/CHANGELOG.md` |

All three carry the canonical `The format is based on [Keep a Changelog]
(https://keepachangelog.com/...)` line inside the CHANGELOG file itself (the
strict test: a KAC-*shaped* changelog with no link does not count). CONTRIBUTING /
docs paths were also grepped for every project that had a changelog; no
additional references surfaced. Of the 83: **3** explicit KAC, **19** real
changelog without a KAC reference (incl. react, vue, flutter, node, axios, d3's
`CHANGES.md`, esbuild, docker/compose), **61** no changelog at the standard path
(mostly GitHub Releases: vscode, tensorflow, ollama, transformers, next.js,
rust's `RELEASES.md`, deno's `Releases.md`).

Method: `gh api search/repositories q='stars:>40000' sort=stars` plus a
`20000..40000` pass, filtered to real software; per repo, looped branches
`{main,master}` × filenames `{CHANGELOG.md, CHANGELOG, CHANGES.md, HISTORY.md,
*.rst}` via `curl`, then grepped each hit for `keepachangelog` / "keep a
changelog". Same lower-bound caveat as §1: code/path search misses references in
non-standard locations.

## 6. Academic / governmental / standards citations

No primary peer-reviewed, governmental, or standards-body citation of Keep a
Changelog was found. KAC is a de-facto community convention; it pairs with SemVer
but neither is an ISO/IETF standard. State this as "no formal standards citation
found" rather than implying one exists.

## Top 5 most citable, defensible stats

1. **6,603 GitHub stars** — exact, GitHub API, trivially reproducible. *(Grows; cite with date.)*
2. **20,640 open-source CHANGELOG files** carry the canonical "based on [Keep a Changelog]" header — GitHub code search, reproducible, a true lower bound. *Strongest single impact number.* *(Grows; re-run before publishing.)*
3. **83,376 code results reference `keepachangelog.com`** — `gh api search/code`. *(Grows.)*
4. **28 languages** — exact, from the repo's `config.rb`. *Most stable.*
5. **Direct parsers: ~140K/mo (PyPI) + ~58K/mo (npm)** — official registry APIs. *(Per-window; cite the date range.)*
