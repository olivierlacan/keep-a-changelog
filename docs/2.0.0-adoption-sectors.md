# Keep a Changelog — sector adoption (named organizations & projects)

Companion to `docs/2.0.0-adoption-research.md`. That file has the **aggregate**
numbers (20,640+ CHANGELOG files carry the canonical header; 83,376 code results
reference `keepachangelog.com`). This file names **specific, verifiable
organizations and projects** across four sectors, for the 2.0.0 References
section.

**Observed: 2026-06-06.** Method: GitHub code search (`gh api search/code`,
authenticated) for the canonical header
`"based on [Keep a Changelog](https://keepachangelog.com"` and for
`keepachangelog` inside named orgs, then **WebFetch of the actual file** to
confirm and capture an exact quote. Web search was used to locate documentation
recommendations; only pages I fetched and read are labelled **Exact**.

**Two signals, kept distinct:**
- **USE** — a CHANGELOG in the org carries the KAC format/header. Weaker if it's
  a single team's file; that's noted.
- **RECOMMEND** — the org's own documentation (CONTRIBUTING, engineering
  handbook, style guide) tells contributors to follow keepachangelog.com. This
  is the stronger signal for the References section, and the part the aggregate
  research was missing.

> **Staleness warning.** Default branches move, files get renamed, and GitHub
> code search indexes only a subset of public repos (so a `0` is *not* proof of
> absence). Counts below are observation-time lower bounds. Re-fetch every cited
> URL right before publishing.

## Debunked / no evidence (commonly assumed, but the header isn't there)

- **Python:** Requests, Click, Poetry, Black, tqdm, Pillow, pre-commit, Astropy, scikit-learn, pydantic, FastAPI.
- **Infra/apps:** Deno, Elasticsearch, Grafana, Home Assistant, Supabase, PostHog, Novu, spotless, lazy.nvim.
- **OWASP MASTG** — keeps a changelog and loosely uses the categories, but the
  file carries **no** keepachangelog.com reference. (OWASP *pytm* does — cite that
  instead.)
- **GitLab** — own git-trailer changelog tooling; no KAC reference.

Takeaway: for marquee standalone OSS the credible story is the **aggregate**
(20,640+ files), plus the handful of genuinely-confirmed names (Textualize, encode).
Don't assert a big project uses KAC without fetching its actual changelog first.

## Reproduce

```sh
# Header count inside an org (lower bound; code search indexes a subset):
gh api -X GET search/code \
  -f q='"based on [Keep a Changelog](https://keepachangelog.com" org:microsoft' \
  --jq '.total_count'

# Any keepachangelog reference (catches CONTRIBUTING/handbook recommendations):
gh api -X GET search/code -f q='keepachangelog org:wikimedia' --jq '.total_count'

# Then WebFetch the .items[].html_url (raw form) and read the actual line.
```

GitHub's REST search rate limit is low (~10 req/min for code search); space calls
out by ~6s or you'll get HTTP 403 rate-limit responses.
