# Keep a Changelog — Verified Citations

## Method and caveats

This document records a skeptical, source-by-source verification of a list of
claimed citations of **Keep a Changelog** (KAC, <https://keepachangelog.com>).
The original list was generated by an LLM (Gemini), which is known to fabricate
academic citations, DOIs, author lists, and the very claim that a source cites
KAC. **Nothing below is taken on the list's word.** A claim is marked
**VERIFIED** only when the actual source was fetched (or a faithful
open-access copy: arXiv preprint, author's institutional PDF, publisher HTML)
and a real reference to "Keep a Changelog" or `keepachangelog.com` was found in
the body text, a footnote, or the bibliography — captured as an exact quote.

- **Access date:** all sources fetched and checked on **2026-06-07**.
- **Staleness caveat:** web pages (industry blogs, The Turing Way, HMC's
  hosted PDF) can change or move after this date. URLs that resolved on the
  access date may rot. Paper DOIs are stable; the open-access PDFs used to read
  full text may move.
- **Open-access substitution caveat:** for paywalled papers (IEEE, ACM) the KAC
  reference was confirmed in the open preprint / author PDF, which is the same
  manuscript but may differ in pagination and reference numbering from the
  final published version of record.

## Verified, citable

These eight sources were read in full text (or faithful open-access copy) and
contain a genuine, quotable reference to Keep a Changelog. They are safe to
list in a public References section.

None. Every source on the list was reachable and its full text or references

## Debunked

- **Venturini, Cogo, Polato, Gerosa & Wiese (2023)**, *I depended on you and
  you broke me* (ACM TOSEM, DOI 10.1145/3576037). The paper is real and the
  authors/title/DOI are correct, **but it does not cite the Keep a Changelog
  standard or keepachangelog.com.** It only recommends maintaining a changelog
  as a general good practice and uses "changelog" / "keep a changelog" as
  generic terms. There is no `keepachangelog` string and no KAC bibliography
  entry anywhere in the manuscript. Gemini's claim that it cites "the KAC
  standard" is fabricated. **Do not cite this as a KAC reference.**

## Author/metadata corrections caught

- **BreakBot:** Gemini listed three authors; the paper has **four** (added
  Jurgen Vinju). Correct arXiv ID is **2111.05132**.
- **HMC:** correct DOI casing is **10.3289/HMC_publ_04**; ten authors, first is
  Pier Luigi Buttigieg.
- **Turing Way:** Gemini's page slug was wrong (404). Real page is
  `reproducible-research/code-documentation/code-documentation-project`.
- **CloudBees:** live URL slug is misspelled `appy-...` on CloudBees' side.
- **Doc Holiday:** domain is `doc.holiday` (no `www`).
