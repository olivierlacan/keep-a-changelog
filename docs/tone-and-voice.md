# Keep a Changelog — tone & voice guide

How we write the manifesto, and the standard the 2.0 rewrite is held to. Many
readers — and every translator — are not native English speakers. Write for them.

## Principles

1. **Lead with the point.** State the recommendation first, then the reason. A
   reader who stops after the first sentence should still get the conclusion.
2. **Plain over clever.** Avoid idioms, metaphors, wordplay, and cultural
   references. They rarely translate and they slow non-native readers.
   *"Don't bolt on a schema"* → *"Don't add a schema."*
3. **Short sentences.** One idea per sentence. Break up long ones. Prefer common
   words and the active voice.
4. **Concise over complete.** Cut words that don't change the meaning. If a
   paragraph repeats a point made elsewhere, delete it.
5. **Consistent terms.** Use the same word for the same thing every time —
   changelog, entry, version, release. Don't vary it for style.
6. **Keep the human voice, sparingly.** A little personality is fine, never at the
   cost of clarity, and not focused on any one person.

## Do / don't

| Don't (idiomatic / verbose) | Do (plain) |
|---|---|
| "take the chore out of it, as long as it doesn't become the chore" | "use it for mechanical tasks" |
| "a happy side effect of writing clearly" | "a benefit of writing clearly" |
| "if your project leans on coding agents" | "if your project uses coding agents" |
| "buries the two changes that mattered under forty that didn't" | "hides the few changes that matter" |
| "trains contributors to paste in a line to pass the check" | "teaches people to add a line to pass the check" |
| phrasal-verb idioms (bolt on, pile up, chase down) | plain verbs (add, accumulate, find) |

## Quick test before publishing a sentence

- Could a fluent-but-non-native reader understand it on the first pass?
- Would a translator have to *rephrase* it rather than translate it?
- Does it say anything a shorter sentence wouldn't?

If any answer is bad, rewrite it.

## Pass it on to maintainers

The manifesto should give maintainers the same advice: many of *their* changelog
readers are not native English speakers either. Recommend plain, concise prose —
clarity over flourish — so a changelog is easy to read and easy to translate. (To
be added to the guidelines as part of the 2.0 restructure.)
